# Session Briefing

세션 간 컨텍스트 연속성을 보장하는 스킬. 작업 결과를 고정 경로에 저장하고, 다음 세션에서 자동 로드.

---

## 발동 조건

| 조건 | 모드 | 행동 |
|------|------|------|
| 작업 완료 | **터보** | 자동실행. 3줄 요약 + 2회 저장 (본파일·_latest.md 모두 덮어쓰기) |
| 10턴↑ 도달 | **터보 + no-op 가드** | 결정·미결·다음 중 ≥1 변화 있을 때만 저장. 변화 없으면 인-컨텍스트 1문단 자가요약만 + 1줄 보고 |
| 형 "이어서" | 로드 | `_latest.md` 1회 read → 재개 |
| 새 세션 시작 | 로드 | `_latest.md` 1회 read → 자동 로드 |
| 형 "풀 브리핑"·"상세 브리핑" | **풀** | 3블록 + 경로 포인터 전부 + 2회 저장 |
| 형 "생략" | — | 1회 스킵 |

이유: UP v37.0 §3 현재성 "장기대화(10턴↑): 핵심 결정·미결·현위치 자가요약"과 일원화. 터보는 체감 지연의 주범인 LLM 추론 토큰을 3블록→3줄로 축약. 풀 모드는 아카이브·회고·중요 마일스톤용으로 보존. no-op 가드는 불필요 저장 차단.

---

## 경로 · 네이밍

```
PROJECT_ROOT = 현재 작업 중인 프로젝트 폴더 (CLAUDE.md/GRAPH.md 소재 폴더 또는 사용자 마운트 폴더)
PATH         = {PROJECT_ROOT}/_session briefing/YYMMDD_작업명.md
LATEST       = {PROJECT_ROOT}/_session briefing/_latest.md
```

### PROJECT_ROOT 판정 (우선순위 + 세션 캐시)
1. 대화 중 명시적으로 지정된 프로젝트 폴더
2. 현재 열린 파일·산출물이 속한 최상위 프로젝트 폴더 (CLAUDE.md 또는 GRAPH.md 존재)
3. 사용자가 마운트한 폴더 루트
4. 위 중 없음 → 1줄 질문으로 확정 후 진행 (폴백 금지)

**세션 캐시:** 같은 세션 내 첫 저장에서 확정된 PROJECT_ROOT를 재사용. 이후 저장은 1~4 재판정 스킵 (사용자가 프로젝트 전환 명시 시에만 갱신).

이유: 매 저장마다 4단계 탐색 = 저장 시간 10~15% 낭비. 세션은 보통 1 프로젝트에 귀속되므로 캐시 적중률 높음.

### 폴더 생성
`_session briefing/` 폴더가 없으면 저장 시 자동 생성. 폴더명은 공백 포함 정확히 `_session briefing` (bash에서는 따옴표·이스케이프 필수).

### TIME_ACQUIRE
YYMMDD = env date에서 추출. 파일명: `YYMMDD_작업명.md` (시간 표기 없음).

이유: 사용자 UP 선호에 따라 파일명에서 시간 제거. 같은 날 동일 작업명은 덮어쓰기(§속도 규칙 참조).

### 네이밍 규칙
- 태그 미사용 — 이미 프로젝트 폴더로 격리되므로 파일명 태그 불필요
- 작업명: 이번 세션 핵심 작업을 한글·영문 혼용 가능, 짧게 (예: `260418_세션브리핑-경로수정.md`)

## 콘텐츠

### 터보 모드 (기본) — 3줄 요약

```
- 결정: [이번 세션에서 확정된 핵심 1~2개 / 한 줄]
- 미결: [보류 + 사유 간단히 / 한 줄. 없으면 "없음"]
- 다음: [다음 세션 첫 행동 1개 + → 파일경로 / 한 줄]
```

포인터(`→ path`)는 **다음액션에만 필수**. 결정·미결에는 변경된 파일이 실제로 있을 때만 1개 자동 포함 (없으면 생략).

이유: LLM 추론 토큰이 체감 지연의 70~80%. 3블록→3줄 축약으로 토큰 60~70% 감소. 다음액션 포인터만 필수인 이유 = "이어서"의 첫 행동만 즉시 실행 가능해야 함.

### 풀 모드 (명시 요청 시) — 3블록 + 포인터 전부

1. **결정사항** — 이번 세션에서 확정된 것 + `→ 관련 파일`
2. **미결** — 보류 중인 것 + 사유 + 외부 대기 항목 + `→ 관련 파일`
3. **다음액션** — 다음 세션 할 일 + `→ 관련 파일`

이유: 회고·중요 마일스톤·외부 대기 항목 많을 때는 풀 트레이스가 필요. "왜 이렇게 결정했지?" 역추적을 위해 결정·미결에도 포인터 필수.

### _latest.md 동기

**저장 = 2회** (날짜별 사본 `YYMMDD_작업명.md` + `_latest.md`, 둘 다 덮어쓰기). 날짜별 사본은 **과거 세션 맥락 보존**용이며 터보 모드에서도 유지.

**_latest.md 정합성 검증:** 로드 시 _latest.md의 mtime과 해당 날짜 브리핑 파일의 mtime 비교. 불일치(날짜 브리핑이 더 신규) → _latest.md 재동기화 후 로드. 일치 → 그대로 사용.

이유: _latest.md 덮어쓰기는 "가장 최근 창문" 역할, 날짜별 사본은 "과거 연대기". 둘 다 있어야 "이어서" 시 빠른 로드 + 2~3세션 전 역추적 동시 가능. 누적 ✗ (덮어쓰기).

## 문서화 전환 게이트

| 모드 | 동작 |
|------|------|
| **터보 (기본)** | 게이트 **스킵** — 즉시 브리핑 저장 |
| **풀** | 합의 3조건 ALL (❶핵심방향 명시적 동의 ❷수정요청 없음 ❸재검토 신호 없음) → "문서로 옮길까요?" 제안 |
| 형 "써줘/저장해" | 즉시 실행 (모드 무관) |
| spine 미합의 | 문서화 ✗ |

이유: 터보는 추가 대화 턴 = 체감 지연. 게이트 스킵으로 1~2턴 절감. 풀 모드에서는 게이트 유지해 과다 저장 방지.

---

## 브리핑 없이 종료 = 컨텍스트 소실

이유: Claude의 세션 간 메모리는 세션 브리핑이 유일한 장치. 저장하지 않으면 다음 세션에서 처음부터 시작해야 한다.

## 속도 규칙

| 규칙 | 효과 |
|------|------|
| **터보 기본** (3줄·포인터 선택) | LLM 추론 토큰 -60~70% (체감 지연 주범 제거) |
| **PROJECT_ROOT 세션 캐시** | 판정 4단계 탐색 -90% (세션 2회차 저장부터) |
| **10턴 no-op 가드** | 변화 없는 저장 스킵 (세션당 20~40% 저장 감소) |
| **터보 시 문서화 게이트 스킵** | 추가 대화 턴 -1~2 |
| 시간 조회 없음 — YYMMDD는 env date만 사용 | bash 호출 1회 제거 (-2~3초). 파일명에 시간 미포함 |
| 디렉토리 존재 확인 생략 — 쓰기 실패 시에만 create | MCP 호출 1회 제거 (-2초) |
| 같은 날 동일 작업명 = 덮어쓰기 (접미사 `_2` 불사용) | list_directory 호출 제거 (-2초). 작업명으로 구분 |
| 저장 = 2회 (날짜별 사본 + _latest.md) 유지 | 과거 맥락 보존 필수. 쓰기 도구 호출은 비용 낮음(5~10%) |
| 로드 = `_latest.md` 1회 read | 정렬·검색 불필요 |

**터보 총 체감:** 풀 모드 대비 저장 1회당 **약 1/3 수준**.

## Gotchas

- **_latest.md는 덮어쓰기 — 누적 ✗:** `_latest.md`에 append 금지. 매 저장마다 덮어쓰기. 과거 기록은 날짜별 사본(`YYMMDD_작업명.md`)이 담당
- **날짜별 사본 생략 = 맥락 소실:** 터보 모드에서도 날짜별 사본은 반드시 저장. 2~3세션 전 역추적 불가 = session-briefing 본래 목적 파괴
- **터보 vs 풀 혼동:** 풀 모드는 "풀 브리핑"·"상세 브리핑"·회고 상황에서만. 일반 작업 완료는 터보 기본
- **PROJECT_ROOT 캐시 오염:** 프로젝트 전환 시 사용자가 명시해야 캐시 갱신. 자동 감지 ✗ (오판정 위험)
- **no-op 가드 남용:** 결정·미결 표현만 바뀌고 실질 변화 없어도 "변화 있음"으로 판정 금지. 실질 차이 없으면 스킵
- **프로젝트 폴더 공백:** `_session briefing` 폴더명에 공백 있음. bash에서는 반드시 따옴표 처리 (`"_session briefing"` 또는 `_session\ briefing`)
- **PROJECT_ROOT 미확정 저장 금지:** 판정 불가 시 볼트 루트·Agent-Ops로 폴백 ✗. 1줄 질문으로 확정 후 진행
- **브리핑 과다:** 일반 대화에서 매번 저장 → 노이즈. 발동 조건표 엄격 준수
- **프로젝트 폴더 접근 확인:** 저장 전 해당 경로 쓰기 가능 여부 확인. 불가 시 마운트 등 대안 탐색
- **같은 날 덮어쓰기 주의:** 파일명에 시간이 없으므로 같은 날 동일 작업명 = 이전 브리핑 덮어씀. 중요한 중간 기록 보존 필요 시 작업명을 구체적으로 (예: `260418_경로수정-1차.md`, `260418_경로수정-검증.md`)
- (실수 발견 시 직접 추가)
