# Session Briefing

세션 간 컨텍스트 연속성을 보장하는 스킬. 작업 결과를 고정 경로에 저장하고, 다음 세션에서 자동 로드.

---

## 발동 조건

| 조건 | 행동 |
|------|------|
| 작업 완료 | 자동실행 (제안 아님) |
| 15턴↑ 도달 | 자동실행 + 1줄 보고 |
| 맥락 임계 (주요작업 3건↑) | 자동실행 + 1줄 보고 |
| 형 "이어서" | 최신 브리핑 자동 로드 + 작업 재개 |
| 새 세션 시작 | PATH에서 최신 브리핑 1건 자동 로드 |
| 형 "생략" | 1회 스킵 |

---

## 경로 · 네이밍

```
PATH: Agent-Ops/_session-briefs/YYMMDD-HHmm_[프로젝트명]_작업명.md
```

### TIME_ACQUIRE (필수)
HHmm 획득시 **bash `date +%y%m%d-%H%M` 실행 필수**. env 날짜만 사용 ✗. 미실행시 시간누락 = FAIL.

이유: env의 날짜는 시·분 정보가 없거나 부정확할 수 있다. bash date가 유일한 정확한 시간 소스.

### 네이밍 규칙
- 시간: 24h 형식
- 프로젝트명 태그: 해당 프로젝트 폴더명 축약
- 프로젝트 미확정 → 태그 생략

---

## 콘텐츠

브리핑에 포함할 항목 (순서 고정):

1. **결정사항** — 이번 세션에서 확정된 것
2. **미결이슈** — 결정 보류 중인 것 + 보류 사유
3. **다음액션** — 다음 세션에서 해야 할 것
4. **의존성** — 외부 대기 항목 (형 판단, 외부 데이터 등)
5. **컨텍스트 포인터** — 관련 파일 경로, 참조 링크

---

## 문서화 전환 게이트

산출물 작성 완료 후 → 브리핑 전에 문서화 전환 제안 가능:

| 조건 | 행동 |
|------|------|
| 합의 3조건 ALL (❶핵심방향 명시적 동의 ❷수정요청 없음 ❸재검토 신호 없음) | "문서로 옮길까요?" 제안 |
| "써줘/저장해" | 즉시 실행 |
| spine 미합의 | 문서화 ✗ |

---

## 브리핑 없이 종료 = 컨텍스트 소실

이유: Claude의 세션 간 메모리는 세션 브리핑이 유일한 장치. 저장하지 않으면 다음 세션에서 처음부터 시작해야 한다.

---

## Gotchas

- **env date 사용:** `date` 환경변수는 시분이 없음 → 파일명 중복·누락. bash date 명령 필수
- **프로젝트 태그 과잉:** "C8-Business-Plan-Meeting-2026-04" 대신 "C8BP" 수준으로 축약
- **브리핑 과다:** 일반 대화에서 매번 저장 → 노이즈. 발동 조건표 엄격 준수
- **이어서 미로드:** "이어서"인데 브리핑 로드 없이 새로 시작 → 형 중복 설명 필요
- (실수 발견 시 직접 추가)
