# session-briefing — Rationale & Gotchas

SKILL.md 본체에서 분리된 설계 근거·예외 상황 전수. 필요 시에만 로드.

---

## 설계 근거

### 왜 터보가 기본인가
UP v37.0 §3 현재성 "장기대화(10턴↑): 핵심 결정·미결·현위치 자가요약"과 일원화. 체감 지연의 70~80%는 LLM 추론 토큰. 3블록→3줄 축약으로 토큰 60~70% 감소. 다음액션 포인터만 필수인 이유 = "이어서"의 첫 행동만 즉시 실행 가능해야 함.

### 왜 풀 모드를 남겨두는가
아카이브·회고·중요 마일스톤·외부 대기 항목 많을 때는 풀 트레이스 필요. "왜 이렇게 결정했지?" 역추적을 위해 결정·미결에도 포인터 필수.

### 왜 볼트에만 저장하는가
세션 마운트 경로에 저장하면 세션 종료 시 소실 or 세션별로 흩어짐 → "이어서"의 전제 파괴. 볼트는 Obsidian·Dropbox로 영속·동기화되므로 세션 간 연속성의 유일한 앵커.

### 왜 PROJECT_ROOT 판정을 2단계로 축약했는가
이전 4단계(명시→CLAUDE.md/GRAPH.md 탐색→마운트 검사→질문)는 첫 저장마다 10~15% 시간 낭비. I/O 탐색보다 1줄 질문이 더 빠름. 캐시 hit 후에는 재판정 ✗.

### 왜 _latest.md 정합성 검증을 제거했는가
이전 규칙: 로드 시 mtime 비교 후 재동기화. 실측: "1회 read" 원칙과 자기모순(최대 3~4회 I/O). 불일치 발생 빈도 낮고, 다음 저장에서 자연 해소. 손실 리스크 < 매번 검증 비용.

### 왜 2회 저장(날짜별 + _latest.md)인가
`_latest.md` 덮어쓰기는 "가장 최근 창문" 역할, 날짜별 사본은 "과거 연대기". 둘 다 있어야 "이어서" 시 빠른 로드 + 2~3세션 전 역추적 동시 가능. 쓰기 도구 호출은 비용 낮음(5~10%).

### 왜 파일명에 시간이 없는가
사용자 UP 선호. 같은 날 동일 작업명 = 덮어쓰기. 중간 기록 보존이 필요하면 작업명을 구체적으로 (예: `260418_경로수정-1차.md`, `260418_경로수정-검증.md`).

### 왜 VAULT 경로 하드코딩 금지인가
`VAULT`는 글로벌 CLAUDE.md의 `MOUNT ::= VAULT`가 유일한 진실원천. SKILL.md·references에 경로 문자열(`/Users/.../ObsidianVault` 등) 박아넣으면 머신·사용자 변경 시 즉사.

### 브리핑 없이 종료 = 컨텍스트 소실
Claude의 세션 간 메모리는 세션 브리핑이 유일한 장치. 저장하지 않으면 다음 세션에서 처음부터 시작.

## Gotchas 전수

- **_latest.md는 덮어쓰기 — 누적 ✗:** `_latest.md`에 append 금지. 매 저장마다 덮어쓰기. 과거 기록은 날짜별 사본(`YYMMDD_작업명.md`)이 담당

- **날짜별 사본 생략 = 맥락 소실:** 터보 모드에서도 날짜별 사본은 반드시 저장. 2~3세션 전 역추적 불가 = session-briefing 본래 목적 파괴

- **터보 vs 풀 혼동:** 풀 모드는 "풀 브리핑"·"상세 브리핑"·회고 상황에서만. 일반 작업 완료는 터보 기본

- **PROJECT_ROOT 캐시 오염:** 프로젝트 전환 시 사용자가 명시해야 캐시 갱신. 자동 감지 ✗ (오판정 위험)

- **세션 마운트 경로에 저장 금지:** `/sessions/.../mnt/outputs` 등 세션 로컬 경로는 세션 종료 시 소실. 반드시 `VAULT` 하위 경로만 사용. 판정 불가 시 1줄 질문으로 확정

- **VAULT 경로 하드코딩 금지:** `VAULT`는 글로벌 CLAUDE.md의 `MOUNT ::= VAULT`가 유일한 진실원천. SKILL.md·references에 경로 문자열 박아넣기 ✗

- **no-op 가드 남용:** 결정·미결 표현만 바뀌고 실질 변화 없어도 "변화 있음"으로 판정 금지. 실질 차이 없으면 스킵

- **프로젝트 폴더 공백:** `_session briefing` 폴더명에 공백 있음. bash에서는 반드시 따옴표 처리 (`"_session briefing"` 또는 `_session\ briefing`)

- **PROJECT_ROOT 미확정 저장 금지:** 판정 불가 시 볼트 루트·Agent-Ops·세션 마운트로 폴백 ✗. 1줄 질문으로 볼트 내 프로젝트 확정 후 진행

- **브리핑 과다:** 일반 대화에서 매번 저장 → 노이즈. 발동 조건표 엄격 준수

- **프로젝트 폴더 접근 확인:** 저장 전 해당 경로 쓰기 가능 여부 확인. 불가 시 마운트 등 대안 탐색

- **같은 날 덮어쓰기 주의:** 파일명에 시간이 없으므로 같은 날 동일 작업명 = 이전 브리핑 덮어씀. 중요한 중간 기록 보존 필요 시 작업명을 구체적으로

- (실수 발견 시 직접 추가)
